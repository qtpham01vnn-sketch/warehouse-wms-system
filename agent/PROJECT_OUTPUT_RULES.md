PROJECT OUTPUT RULES – PHUONGNAM APP TA2

1. Cấu trúc output bắt buộc

Mỗi project tạo ra phải có:

* src/
* public/
* README.md
* package.json
* .env.example

---

2. Naming (Quy tắc đặt tên)

* Tên project: không dấu, chữ thường
* Format:

  * app-kho
  * app-mmtb
  * app-iso
  * landing-xxx

---

3. README bắt buộc

README phải có:

* Giới thiệu app
* Hướng dẫn chạy
* Cấu trúc thư mục
* Công nghệ sử dụng
* Hướng dẫn deploy

---

4. Database (Supabase)

* Có schema rõ ràng
* Có bảng chính
* Có quan hệ dữ liệu
* Field đặt tên rõ ràng

---

5. UI & Pages

* Có Dashboard
* Có danh sách (List)
* Có Form nhập liệu
* Có trang chi tiết

---

6. Tách module rõ ràng

* Không viết 1 file lớn
* Chia theo:

  * pages
  * components
  * services

---

7. Chuẩn hoá output

* Code rõ ràng
* Có comment nếu cần
* Không dư thừa
* Không lỗi

---

8. Có thể deploy

* Chạy được ngay sau khi clone
* Không lỗi dependency
* Có hướng dẫn rõ ràng

---

Nguyên tắc:

* Mỗi output là 1 project hoàn chỉnh
* Không output rời rạc
* Không thiếu file quan trọng
