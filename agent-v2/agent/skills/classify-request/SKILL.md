# 🧠 Skill: Classify Request

Phân loại yêu cầu của người dùng để xác định luồng công việc (workflow) phù hợp nhất.

## 🎯 Mục tiêu
Đảm bảo mọi yêu cầu đều được gán vào một trong các danh mục (categories) đã định nghĩa.

## 📋 Danh mục (Categories)
- **app**: Yêu cầu tạo mới hoặc chỉnh sửa ứng dụng web/phần mềm.
- **landing**: Yêu cầu tạo trang landing page, portfolio hoặc trang giới thiệu.
- **prompt**: Yêu cầu tối ưu hóa câu lệnh, tạo hệ thống prompt chuyên biệt.
- **full_package**: Yêu cầu trọn gói bao gồm cả thiết kế, code và tài liệu.

## 🛠 Cách hoạt động
1. Phân tích nội dung tin nhắn của người dùng.
2. Kiểm tra các từ khóa quan trọng (keywords).
3. Gán thẻ (tag) tương ứng cho giai đoạn triển khai tiếp theo.
